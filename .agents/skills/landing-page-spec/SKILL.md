# Landing Page Spec

Use this skill for landing-page behavior, navigation, or theme-persistence changes.

- Read `docs/specs/landing-page/requirements.md` before changing user-visible behavior.
- Read `docs/specs/landing-page/design.md` before changing architecture or page flow.
- Treat those documents as the source of truth for acceptance criteria and implementation intent.
- If the code and docs diverge, note the mismatch explicitly instead of silently choosing one.
