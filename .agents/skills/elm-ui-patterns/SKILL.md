# Elm-UI Patterns

Use this skill for `.elm` view work.

- Build layouts with `elm-ui` primitives instead of mixing in custom HTML/CSS structures.
- Pull colors and shared styling from the theme system.
- Use the repo's responsive utilities for font sizes, padding, spacing, and grid sizing.
- Keep headers, controls, and grid layouts structurally consistent with the existing app.
- Reserve CSS for cases `elm-ui` cannot reasonably express.
- Preserve accessibility with semantic labels and keyboard-friendly interactions.
