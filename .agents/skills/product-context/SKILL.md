# Product Context

Use this skill before making feature-level decisions.

- The repo is a demo Elm application centered on a human-versus-AI tic-tac-toe experience.
- The tic-tac-toe game uses game-theory search and worker-based computation to keep UI interactions responsive.
- The robot game is a separate one-person movement demo with directional controls and animation behavior.
- Shared themes and a theme/style-guide surface are part of the product story.
- Hash routing is part of the user-facing experience, not just an implementation detail.
- When requirements are ambiguous, defer to `docs/specs/`.
